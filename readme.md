# service generator

## 목표
- **생산성 극대화하여 간단한 서비스 찍어내기 (수익모델은 광고)**
- 아이디어 참신해야 함
  - idea generator 예정
- 디자인 이뻐야 함
  - 디자인 검수 과정 잘해야함
- 간단해야 함
  - phase 1수준으로만 진행
- 개발 퀄리티 및 유지보수는 중요하지 않음 (검증 케이스만 잘 돌아가면 됨)
  - 코드에 관여 x
- 과금 최대한 안되게
  - 프레임워크 없이 직접 개발.
  - 디자인 프로토타입 없이 이미지로만.
- 1차 목표는 사람 개입 최소화 상관없이 서비스 만드는 것.

## 실행 환경
- 로컬 환경 claude code로 모든 단계를 진행한다.
- 각 단계 트리거는 모두 사용자에 의해 수동 트리거 된다.
- 각 단계에 맞는 에이전트 md파일을 잘 관리하고 claude.md에서 지침을 통해 라우팅을 진행한다.
- 각 에이전트 폴더가 있고, 결과물은 모두 디렉토리 내에 저장하고 다른 에이전트와 공유한다.

## service generator flow
1. 아이디어 구체화
   1. 사람이 아이디어를 가지고 온다.
   2. ai와 대화를 통해 아이디어를 구체화 시키고 문서로 만든다.
      1. 기존에 아이디어 비슷한 서비스 있는지, 차별점 도출 필요
      2. 종합적으로 성공 가능성 있는지 검토 받음
      3. 핵심적인 기능을 정의한다.
   3. 만들어진 문서에 대해 ai에게 피드백 하면서 문서 최종본을 완성시킨다.
2. 기능 명세
   1. 아이디어 구체화 문서를 기반으로 ai가 기능 명세서를 작성한다.
      1. 유스케이스 명세
      2. 엣지케이스 명세
      3. 테스트케이스 명세
   2. 만들어진 문서에 대해 ai에게 피드백하면서 문서 최종본을 완성시킨다.
3. 디자인 명세 단계
   1. 앞에서 만들어진 문서를 보고 ai가 디자인 컨셉을 뽑는다
      1. 디자인 컨셉은 샘플 화면
      2. design.md 생성
   2. 사람이 ai에게 피드백하면서 디자인 컨셉을 수정한다.
   3. 기능 명세서를 기반으로 디자인 화면을 명세한다.
      1. stitch sdk 써서 화면 이미지 추출
4. supabase 구축
   1. 최종 설계 문서를 보고 ai가 supabase 구축을 한다.
      2. supabase 명세서 필요
5. client 소스코드 개발
   1. 앞서 만들어진 모든 문서 및 리소스를 보고 개발한다.
      2. 소스코드 작업 내용 정리 필요
   2. 광고 추가
6. qa 진행
   1. ai가 기능 명세서를 기반으로 qa 테스트 항목을 만들고 진행하고 결과서를 저장한다.
   2. 실패 결과에 대해서 소스 코드 수정을 요청한다.
7. 직접 테스트 진행
   1. 사람이 직접 테스트 진행한다.
   2. 실패 결과에 대해서 소스 코드 수정을 요청한다.
9. 배포
